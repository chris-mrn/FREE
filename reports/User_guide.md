# User Guide — TorchCFM Speed-Adaptive Flow Matching

This guide covers how to set up the environment, run every type of experiment, and use the Gatsby cluster.

---

## What this repo does

This repo studies **speed-adaptive time-sampling for flow matching** on CIFAR-10 and 2D toy datasets. The core idea: instead of sampling training time `t` uniformly, sample it proportionally to the speed of the probability path — spending more training compute in regions where the flow changes quickly. Three strategies are implemented:

| Strategy | How `t` is sampled |
|---|---|
| `uniform` | `t ~ Uniform(0, T_MAX)` — baseline |
| `weighted` | `t ~ q(t) ∝ v_t`, precomputed from a trained model |
| `curriculum` | Starts uniform, estimates speed online at a milestone step, then blends to speed-adaptive |

Two interpolation geometries are supported (`--path linear` and `--path spherical`), as well as a self-interpolant variant where both source and target are data images (no Gaussian noise).

## Running experiments

All commands below assume you are in the repo root and have set `PYTHONPATH=.`. The training entry point is `train.py`. **Checkpoints auto-resume** — re-running the same command with the same `--out_dir` picks up where it left off. Pass `--resume disabled` to restart from scratch.

### 1. Linear FM — CIFAR-10, uniform t (baseline)

```bash
PYTHONPATH=. python train.py \
    --path linear --dataset cifar10 \
    --coupling independent --t_mode uniform \
    --total_steps 400001 --batch_size 128 \
    --eval_every 20000 --fid_samples 10000 \
    --out_dir outputs/cifar10_linear
```

### 2. Spherical FM — CIFAR-10, uniform t

```bash
PYTHONPATH=. python train.py \
    --path spherical --dataset cifar10 \
    --coupling ot --t_mode uniform \
    --total_steps 400001 --batch_size 128 \
    --eval_every 20000 --fid_samples 10000 \
    --out_dir outputs/cifar10_spherical
```

### 3. Weighted t-sampling from precomputed speed

Requires precomputed speed `.npy` files in `--speed_dir` (generated by scripts in `speed/`):

```bash
PYTHONPATH=. python train.py \
    --path spherical --dataset cifar10 \
    --coupling ot --t_mode weighted \
    --speed_type ot --speed_dir outputs/cifar10_spherical \
    --total_steps 400001 --batch_size 128 \
    --out_dir outputs/cifar10_sph_weighted
```

### 4. Curriculum training — online speed estimation

Trains uniform until `--curriculum_start`, then estimates speed from the EMA model and blends over `--curriculum_blend` steps:

```bash
PYTHONPATH=. python train.py \
    --path spherical --dataset cifar10 \
    --coupling ot --t_mode curriculum \
    --speed_type ot \
    --curriculum_start 100000 --curriculum_blend 25000 \
    --total_steps 400001 --batch_size 128 \
    --out_dir outputs/cifar10_sph_curriculum
```

### 5. Multi-GPU DDP (same as above, 3 GPUs)

```bash
PYTHONPATH=. torchrun --nproc_per_node=3 \
    --nnodes=1 --master_addr=localhost --master_port=29502 \
    train.py \
    --path spherical --dataset cifar10 \
    --coupling ot --t_mode curriculum \
    --speed_type ot \
    --curriculum_start 100000 --curriculum_blend 25000 \
    --total_steps 400001 --batch_size 128 \
    --ddp --out_dir outputs/cifar10_sph_ot_curriculum
```

### 6. 2D toy experiments

```bash
# Single dataset
PYTHONPATH=. python train.py \
    --path linear --dataset 8gaussians \
    --t_mode uniform --total_steps 50000 \
    --out_dir outputs/2d_8gaussians

# Available datasets: 8gaussians, 40gaussians, moons, circles, checkerboard
```

### 7. Self-FM — data-to-data interpolant

The self-FM model uses real CIFAR-10 images as both source and target. Training script is separate from `train.py`:

```bash
PYTHONPATH=. python examples/images/cifar10/training/train_self_fm.py \
    --total_steps 200001 --batch_size 128 \
    --speed_step 100000 \
    --out_dir outputs/cifar10_self_fm
```

Set `--speed_step 999999` to disable the curriculum and train with uniform `t` throughout (ablation).

### 8. Latent FM — flow matching in VAE latent space

Step 1: train a VAE on CIFAR-10 (produces `outputs/cifar10_vae/vae_ema_final.pt`):

```bash
PYTHONPATH=. python scripts/train_vae_cifar10.py \
    --out_dir outputs/cifar10_vae --total_steps 100000
```

Step 2: train the latent FM (requires the VAE checkpoint from step 1):

```bash
PYTHONPATH=. torchrun --nproc_per_node=1 \
    scripts/train_fm_latent_linear_ddp.py \
    --vae_ckpt   outputs/cifar10_vae/vae_ema_final.pt \
    --latent_stats outputs/cifar10_vae/latent_stats.pt \
    --out_dir    outputs/cifar10_latent_linear \
    --total_steps 200000
```

## Evaluating trained models

### NFE sweep (standard FM)

Sweeps over a list of Euler steps and writes `nfe_fid_table.csv`:

```bash
PYTHONPATH=. python scripts/eval_fid_nfe.py \
    --ckpt_dir  outputs/cifar10_spherical/checkpoints \
    --out_dir   outputs/cifar10_spherical \
    --metrics_csv outputs/cifar10_spherical/metrics.csv \
    --nfe_list  10 35 50 100 200 \
    --n_samples 10000 \
    --spherical   # omit this flag for linear path
```

### NFE sweep (self-FM)

```bash
PYTHONPATH=. python scripts/eval_self_fm_nfe.py \
    --ckpt_dir  outputs/cifar10_self_fm/checkpoints \
    --out_dir   outputs/cifar10_self_fm \
    --nfe_list  5 10 20 35 50 100 200 \
    --n_samples 10000
```

### Reparameterised divergence (self-FM analysis)

Measures whether the FR arc-length reparametrisation achieves constant-speed flow:

```bash
PYTHONPATH=. python scripts/compute_reparam_div.py \
    --ckpt    outputs/cifar10_self_fm/checkpoints/ema_step_0200000.pt \
    --speed_t outputs/cifar10_self_fm/fr_t_grid_step100000.npy \
    --speed_v outputs/cifar10_self_fm/fr_speed_step100000.npy \
    --out_dir outputs/cifar10_self_fm
```

### Fisher-Rao speed computation (latent flows)

Estimate Fisher-Rao speed on a trained latent FM model using Hutchinson divergence estimator (with configurable probes):

```bash
PYTHONPATH=. python scripts/compute_latent_fr_speed.py \
    --ckpt outputs/cifar10_latent_linear/checkpoints/ckpt_step_0200000.pt \
    --latent_stats outputs/cifar10_vae/latent_stats.pt \
    --out_dir outputs/latent_speed_fr \
    --speed_n_t 50 \
    --speed_B 512 \
    --speed_epochs 3 \
    --speed_hutch 5 \
    --channel_mult 1 2
```

**Important:** `--channel_mult 1 2` must match the model architecture used during training. For standard latent FM (trained with `train_fm_latent_linear_ddp.py`), this is the default.

Outputs:
- `fr_speed.npy` — speed curve v_t^FR on the grid
- `fr_weighting.npy` — weighting w(t) = L / v_t for curriculum training
- `t_grid.npy` — corresponding time grid

The script loads the VAE encoder to collect reference latents from CIFAR-10, then estimates divergence using 5 Hutchinson probes (configurable with `--speed_hutch`).

#### Visualize the speed profile

After computing Fisher-Rao speed, create a 3-panel plot showing speed, weighting, and arc-length schedule:

```bash
PYTHONPATH=. python scripts/plot_latent_speed.py \
    --speed_dir outputs/latent_speed_fr \
    --out_dir outputs/latent_speed_fr \
    --smooth_sigma 0.05
```

**Outputs:**
- **Panel A: Speed v_t^FR** — Fisher-Rao speed showing where the flow changes rapidly
- **Panel B: Weighting w(t) = L/v_t** — inverse-speed weighting (raw and smoothed)
- **Panel C: Arc-length schedule α(t)** — reparameterisation function that maps uniform sampling in α-space to weighted sampling in t-space

The plot includes summary statistics: speed range, weighting range, coefficient of variation, and sampling density characteristics.

## Using Pre-Trained Weights

This repo ships with a pre-trained FM checkpoint useful for speed estimation and other downstream tasks.

### Pre-trained FM weights

**File:** `weights/fm_cifar10_weights_step_80000.pt` (546 MB)

An early CIFAR-10 flow matching model (linear path, independent coupling, step 80K). Useful for:
- Speed estimation without training a full model
- Fine-tuning or continued training
- Computing inference statistics

#### Continue training from pre-trained FM weights

Re-run the same training command with `--resume auto`. The framework will auto-detect and resume from the latest checkpoint in `--out_dir`. To start fresh with pre-trained weights as initialization:

```bash
# Initialize a new run from the pre-trained weights
PYTHONPATH=. python train.py \
    --path linear --dataset cifar10 \
    --out_dir outputs/cifar10_finetuned \
    --total_steps 600001 \
    --batch_size 128
```

Then manually copy the pre-trained weights into the new output directory:

```bash
mkdir -p outputs/cifar10_finetuned/checkpoints
cp weights/fm_cifar10_weights_step_80000.pt outputs/cifar10_finetuned/checkpoints/ckpt_step_00080000.pt
```

Re-run the training command — it will auto-resume from step 80K.

#### Compute statistics on pre-trained FM

Generate samples and compute FID without retraining:

```bash
PYTHONPATH=. python scripts/eval_fid_nfe.py \
    --ckpt_dir weights \
    --out_dir outputs/pretrained_eval \
    --nfe_list 10 35 50 100 200 \
    --n_samples 10000 \
    --metrics_csv outputs/pretrained_eval/metrics.csv
```

#### Use pre-trained FM for speed estimation

Estimate speed curves from the pre-trained model:

```bash
PYTHONPATH=. python speed/cifar10_speed.py \
    --ckpt weights/fm_cifar10_weights_step_80000.pt \
    --out_dir outputs/pretrained_speed \
    --speed_type ot
```

Then use the computed speed for curriculum or weighted t-sampling on a new run.

### Pre-trained VAE weights

**File:** `outputs/cifar10_vae/vae_ema_final.pt` (generated by `scripts/train_vae_cifar10.py`)

#### Train latent FM with a pre-trained VAE

If you have already trained a VAE, simply pass its checkpoint to the latent FM training script:

```bash
PYTHONPATH=. torchrun --nproc_per_node=1 \
    scripts/train_fm_latent_linear_ddp.py \
    --vae_ckpt   outputs/cifar10_vae/vae_ema_final.pt \
    --latent_stats outputs/cifar10_vae/latent_stats.pt \
    --out_dir    outputs/cifar10_latent_linear \
    --total_steps 200000 --batch_size 128
```

The VAE is frozen during FM training; only the velocity field is updated.

#### Continue latent FM training

Auto-resume from checkpoints:

```bash
PYTHONPATH=. torchrun --nproc_per_node=1 \
    scripts/train_fm_latent_linear_ddp.py \
    --vae_ckpt   outputs/cifar10_vae/vae_ema_final.pt \
    --latent_stats outputs/cifar10_vae/latent_stats.pt \
    --out_dir    outputs/cifar10_latent_linear \
    --total_steps 400000  # extend from 200k to 400k
```

Pass `--resume disabled` to restart from scratch.

## Understanding outputs

Each training run writes to its `--out_dir`:

| File / Directory | Contents |
|---|---|
| `checkpoints/ckpt_step_*.pt` | Full checkpoint: model, EMA, optimizer, scheduler, curriculum state |
| `samples/step_*.png` | 8×8 grids of generated images, saved every `--eval_every` steps |
| `loss.csv` | Per-100-step loss log: `step, loss_raw, loss_ema, t_phase` |
| `metrics.csv` | Per-eval FID / KID / IS: `step, fid, kid_mean, kid_std, is_mean, is_std` |
| `nfe_fid.csv` | NFE sweep at end of training |
| `speed_t_grid_step*.npy` / `speed_v_t_step*.npy` | Speed profiles estimated during curriculum |

Training always resumes automatically from the latest checkpoint when you re-run the same command. The curriculum phase (0=uniform, 1=blending, 2=pure speed-adaptive) is stored in the checkpoint and restored on resume.

## Cluster usage

### Available GPU partitions

| Partition | GPU types | Max time | Notes |
|---|---|---|---|
| `gpu_lowp` | H100 (4–8×), H200 (8×), L40S (7–8×) | 10 days | Best GPUs; low-priority queue |
| `a100` | A100 (4× per node) | 10 days | Dedicated A100 nodes |
| `gpu` | A4500, A100, L40S, RTX 4000, P5000, RTX 2080 | 10 days | Mixed bag; generally available |
| `gatsby_ws` | RTX 4500 (1× per node) | 10 days | Workstation nodes; low VRAM (24 GB) |

**GPU hierarchy** (prefer higher): H200 > H100 > A100 > L40S > A4500/RTX4500 > P5000/RTX 2080

Check live availability before submitting:

```bash
sinfo -o "%.18R %.6t %N %G" | grep -E "gpu|a100"
```

### Submitting batch jobs

Ready-made Slurm scripts live in `slurm/`. Submit with:

```bash
sbatch slurm/train_linear_cifar10.sh          # Linear FM, 1× H100
sbatch slurm/train_spherical_ot_curriculum.sh  # Spherical + OT curriculum, 3× H100
sbatch slurm/train_vae_cifar10.sh              # VAE training
sbatch slurm/train_latent_fm_a100.sh           # Latent FM, A100
sbatch slurm/train_self_fm_uniform_ablation.sh # Self-FM uniform ablation
sbatch slurm/eval_sph_fm_nfe.sh                # NFE sweep (spherical)
sbatch slurm/eval_self_fm_uniform_150k.sh      # Self-FM NFE sweep
sbatch slurm/compute_reparam_div.sh            # Reparameterised divergence
sbatch slurm/reparam_div_2d.sh                 # 2D reparameterised divergence sweep
```

Monitor your jobs:

```bash
squeue -u $USER                  # list your running/pending jobs
squeue -u $USER -o "%.10i %.20j %.8T %.10M %.6D %R"  # more detail
scancel <JOBID>                  # cancel a job
```

Logs are written to the path specified in `#SBATCH --output` inside each script (e.g. `outputs/<run>/train_<jobid>.log`).

### Checking GPU memory before submitting

On an interactive node, to check how much VRAM is free:

```bash
nvidia-smi --query-gpu=name,memory.total,memory.free --format=csv
```

A standard CIFAR-10 training run (`--num_channel 128`, batch 128) uses roughly **8–10 GB** per GPU. A 3-GPU DDP run needs 3 GPUs with ≥10 GB each.
